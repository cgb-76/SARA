  <action>
    Create `/home/george/Projects/llm-wiki-gsd/install.sh` as an executable bash script.

    The script must implement all of the following in order:

    **Header and argument parsing:**
    - Shebang: `#!/usr/bin/env bash`
    - `set -euo pipefail`
    - Parse flags in a loop: `--backup` sets BACKUP=true; `--force` sets FORCE=true; `--target <dir>` sets TARGET_DIR to next arg; `--help` prints usage and exits 0; any unknown flag exits 1 with usage message
    - Default: BACKUP=false, FORCE=false, TARGET_DIR=$PWD

    **Guard: script must run from inside a git repo (D-03):**
    - Check that TARGET_DIR contains a `.git` directory
    - If `.git` is absent: print `Error: install.sh must be run inside a git repository. SARA pipeline commands depend on git commits.` to stderr and exit 1

    **Locate source skills:**
    - SCRIPT_DIR is the directory containing install.sh (use `$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)`)
    - SOURCE_SKILLS_DIR="$SCRIPT_DIR/.claude/skills"
    - Glob all `sara-*` directories under SOURCE_SKILLS_DIR (D-04 — dynamic, no hardcoded list)
    - If no sara-* dirs found: print `Error: no sara-* skill directories found in $SOURCE_SKILLS_DIR` to stderr and exit 1

    **Create target skills directory (Claude's Discretion):**
    - TARGET_SKILLS_DIR="$TARGET_DIR/.claude/skills"
    - `mkdir -p "$TARGET_SKILLS_DIR"`

    **Per-skill install loop:**
    For each sara-* source directory:
    1. Extract skill name: `skill_name=$(basename "$src_skill_dir")`
    2. Determine dest dir: `dest_skill_dir="$TARGET_SKILLS_DIR/$skill_name"`
    3. Extract source version: `src_ver=$(grep "^version:" "$src_skill_dir/SKILL.md" 2>/dev/null | awk '{print $2}')`
       - If src_ver is empty: treat as "0.0.0"
    4. If dest SKILL.md exists: extract installed version `inst_ver=$(grep "^version:" "$dest_skill_dir/SKILL.md" 2>/dev/null | awk '{print $2}')`; if empty treat as "0.0.0"
    5. **Downgrade check (D-09):** If dest SKILL.md exists and FORCE is not true:
       - Compare: `older=$(printf '%s\n%s\n' "$src_ver" "$inst_ver" | sort -V | head -1)`
       - If `[ "$older" = "$src_ver" ] && [ "$src_ver" != "$inst_ver" ]`: source is older — downgrade scenario
       - Print to stderr: `Warning: source version ($src_ver) is older than installed version ($inst_ver) for $skill_name — skipping. Use --force to override.`
       - `continue` (skip this skill)
    6. **Backup (D-05):** If BACKUP=true and dest SKILL.md exists: `cp "$dest_skill_dir/SKILL.md" "$dest_skill_dir/SKILL.md.bak"`
    7. **Copy:** `mkdir -p "$dest_skill_dir"` then `cp -r "$src_skill_dir/." "$dest_skill_dir/"`
    8. Append skill_name to INSTALLED list

    **Post-install output (D-07):**
    - If INSTALLED list is empty: print `No skills were installed.` and exit 0
    - Print `Installed skills:` then each skill name on its own line (one per line, no bullets)
    - Print blank line
    - Print `Next: open Claude Code in this directory and run /sara-init`

    **Make executable:**
    After writing the file, run: `chmod +x /home/george/Projects/llm-wiki-gsd/install.sh`
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user→install.sh | User provides --target path; script copies files there |
| install.sh→filesystem | Script reads from source repo and writes to target project |
| curl pipe (distribution) | Users may pipe the script from a raw URL before reading it |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03 | Tampering | install.sh --target argument | mitigate | TARGET_DIR resolved via cd+pwd; no eval; cp uses explicit src and dest paths; no path constructed via user input concatenation beyond the validated target dir |
| T-04-04 | Elevation of Privilege | install.sh executed as current user | accept | Script runs as the invoking user; no sudo; installs only to user-controlled project directories |
| T-04-05 | Spoofing | curl-pipe execution (users pipe from raw URL) | accept | Out of scope for this plan — script itself is safe; distribution security depends on HTTPS and GitHub's integrity guarantees |
| T-04-06 | Tampering | Overwrite of existing customised SKILL.md | mitigate | --backup flag preserves SKILL.md.bak; downgrade warning gives user a chance to review before overwriting an upgraded version |
| T-04-07 | Denial of Service | install.sh writing to unintended path | mitigate | Script checks for .git in TARGET_DIR before writing anything; no write occurs outside TARGET_DIR/.claude/skills/ |
</threat_model>
