<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| {filename} argument → filesystem path | User-supplied filename is interpolated into directory and file paths |
| Counter derivation → ID assignment | Filesystem glob output determines the next ID; injection or collision is possible if directory names are malformed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-02-01 | Tampering | {filename} path traversal | mitigate | Step 1 validation (no / or ..) is preserved exactly from current skill; acceptance criteria verify the note is retained |
| T-17-02-02 | Elevation of Privilege | mkdir -p path construction | mitigate | {new_id} is always constructed from {type_key} (hardcoded) and the filesystem-derived counter — never from user input directly; {filename} is only used in source_path construction after validation |
| T-17-02-03 | Spoofing | Counter derivation ID collision | accept | The grep sorts and takes tail -1, then increments — two concurrent ingest calls could theoretically derive the same ID; accepted because SARA is a solo-user tool with no concurrent invocation |
| T-17-02-04 | Information Disclosure | grep -rh output | accept | .sara/pipeline/ is a local project directory; no sensitive data beyond project-local source paths |
</threat_model>
