<objective>
Activate the `tags: []` schema field on all wiki artifact pages by implementing D-08 — the whole-wiki LLM tag curation check — as Step 6 of sara-lint.

D-08 runs after the Step 5 per-finding loop on every sara-lint invocation. It performs a two-phase pass:
1. **Vocabulary derivation**: reads all artifact pages (frontmatter + first body section) and derives an emergent set of lowercase kebab-case tag strings representing conceptual themes across the wiki corpus.
2. **Vocabulary approval gate**: presents the derived tags to the user via AskUserQuestion. User can Approve, Edit, or Skip.
3. **Assignment pass**: for each artifact page, reads the full page and determines which approved vocabulary tags apply.
4. **Batch write + atomic commit**: writes updated `tags: [...]` frontmatter to all tagged pages and commits as a single `fix(wiki): update tags via sara-lint D-08` commit.

Purpose: Tags are a concept-level cross-reference axis — distinct from `related[]` (which links specific artifacts). A tag like `data-governance` can link a requirement, a decision, and a risk by shared theme even if none directly reference each other. Tags are the primary filter axis for the future `/sara-query` command.

Output: Updated `.claude/skills/sara-lint/SKILL.md` with revised objective line and new Step 6 block.
