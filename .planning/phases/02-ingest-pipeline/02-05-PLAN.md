Create `.claude/skills/sara-extract/SKILL.md` as a new file. Structure: YAML frontmatter → `<objective>` → `<process>` → `<notes>`.

**Frontmatter:**
```yaml
---
name: sara-extract
description: "Present proposed wiki artifacts for per-artifact approval before any wiki writes"
argument-hint: "<N>"
allowed-tools:
  - Read
  - Write
  - Bash
  - AskUserQuestion
---
```

**Objective:** Two sentences: (1) Reads source document and discussion notes, checks existing wiki pages for deduplication, and presents each proposed artifact (with source-quote evidence) for user approval via accept/reject/discuss. (2) Approved artifacts are written to `extraction_plan` in `pipeline-state.json`; no wiki files are created or modified by this skill.

**Process — 5 steps:**

**Step 1 — Stage guard and item lookup**

Read `.sara/pipeline-state.json`.

Validate `$ARGUMENTS`: must be a positive integer. If not: output `"Usage: /sara-extract <N> where N is a positive integer pipeline item number."` and STOP.

Find item with key `"{N}"` in `items`. If not found: output `"No pipeline item {N} found. Run /sara-ingest to register a new item, or run /sara-ingest with no arguments to see the full pipeline status."` and STOP.

Check `items["{N}"].stage`. Expected: `"extracting"`.
If actual stage != `"extracting"`:
  Output: `"Item {N} ({id}) is in stage '{actual_stage}'. Run /sara-extract N only when stage is 'extracting'. Re-run /sara-discuss N if you need to revisit the discussion."`
  STOP.

Store `{item}` = `items["{N}"]`.

**Step 2 — Load source, discussion notes, and dedup context**

Read `raw/input/{item.filename}` — the source document.

`{discussion_notes}` = `items["{N}"].discussion_notes` (already in memory from Step 1 read).

Read `wiki/index.md` HERE (at this step, not earlier). Reading index here ensures it is fresh — it may have been updated by `/sara-add-stakeholder` during the preceding `/sara-discuss` session. (See notes — Pitfall 4.)

**Step 3 — Generate artifact proposals with dedup check**

Using the source document, `{discussion_notes}`, and the current `wiki/index.md`:

For each extractable topic in the source:
  Search `wiki/index.md` for an existing entity with a matching or similar title/description.
  If a matching entity is found in the index:
    → Propose action: `"update"`, set `existing_id` to the matching entity's ID, set `change_summary` to what needs to change.
  If no matching entity:
    → Propose action: `"create"`, set `id_to_assign` to `"{TYPE}-NNN"` (placeholder; real ID assigned at update time).

Every artifact must include `source_quote`: an exact verbatim passage from the source document that supports this artifact. Do not propose any artifact without a source quote.

Produce a list of proposed artifact objects following this schema:
```json
{
  "action": "create" | "update",
  "type": "requirement" | "decision" | "action" | "risk",
  "id_to_assign": "{TYPE}-NNN",
  "existing_id": "{existing entity ID}",
  "title": "{title}",
  "source_quote": "{exact verbatim text from source}",
  "raised_by": "{STK-NNN or empty string}",
  "related": [],
  "change_summary": "{for update only}"
}
```

Fields `id_to_assign` and `existing_id` are mutually exclusive: use `id_to_assign` for action=create, `existing_id` for action=update. Omit the unused field or set it to `""`.

**Step 4 — Per-artifact approval loop**

Initialize `approved_artifacts = []`.

For each proposed artifact at index `{artifact_index}` (starting at 1):

  Present artifact as plain text before the AskUserQuestion call:
  ```
  --- Artifact {artifact_index} ---
  Type:   {type}
  Title:  {title}
  Action: CREATE new {TYPE}-NNN  /  UPDATE {existing_id}
  Source: "{source_quote}"
  [If update] Change: {change_summary}
  ```

  Then call AskUserQuestion:
  ```
  header: "Artifact {N}"    (for artifact_index 1–9: "Artifact {N}" = 10 chars, safe)
                             (for artifact_index 10+: "Item {N}" = 7 chars, safe)
  question: "Accept, reject, or discuss artifact {artifact_index}?"
  options: ["Accept", "Reject", "Discuss"]
  ```

  If "Accept":
    Append the artifact object to `approved_artifacts`.
    Continue to next artifact.

  If "Reject":
    Skip this artifact. Continue to next artifact.

  If "Discuss":
    Output as plain text: `"What would you like to change about this artifact?"`
    Wait for user's reply. (Plain text wait — NOT another AskUserQuestion. Freeform rule.)
    Incorporate the user's correction into the artifact object (update title, source_quote, type, or change_summary as directed).
    Re-present the revised artifact as plain text (same format as above).
    Loop back to the AskUserQuestion for this artifact.
    Repeat until the user selects "Accept" or "Reject".

After all artifacts have been Accept or Reject resolved: proceed to Step 5.

If `/sara-extract N` is re-run on an item already in `extracting` stage (possible if previous session was interrupted): re-run the full loop with freshly generated proposals. The wiki has not been written yet; it is safe to re-run.

**Step 5 — Write extraction plan and advance stage**

Read `.sara/pipeline-state.json`.
Update `items["{N}"]`:
  - `stage`: `"approved"`
  - `extraction_plan`: `[...approved_artifacts array...]`
Write the modified JSON back to `.sara/pipeline-state.json` using the Write tool.
Do NOT use Bash jq or sed.

Output a summary table:
```
## Extraction Plan — Item {N}

{count_accepted} artifacts accepted / {count_rejected} rejected.

| # | Action | Type        | Title                              |
|---|--------|-------------|------------------------------------|
| 1 | CREATE | requirement | API rate limiting per tenant       |
| 2 | UPDATE | decision    | Auth token expiry policy           |

Run /sara-update {N} to write approved artifacts to the wiki.
```

**Notes block:**
- source_quote is MANDATORY for every artifact. An artifact without a source quote must not be proposed. This is the evidence trail.
- wiki/index.md is re-read at Step 2 (dedup step), not at skill entry, to catch index updates from /sara-add-stakeholder in the same session (Pitfall 4 guard).
- AskUserQuestion header hard limit is 12 characters. Use "Artifact {N}" for N=1–9 (10 chars). Use "Item {N}" for N=10+ (7 chars). Never exceed 12.
- When user selects "Discuss": use plain text output and wait for reply (freeform rule). Do NOT use another AskUserQuestion. Resume AskUserQuestion only after processing the correction.
- extraction_plan is written ONLY after the full loop completes. If session resets mid-loop, the extraction_plan in pipeline-state.json will be empty or partial. Re-run /sara-extract N to start a fresh loop — the wiki has not been written yet, so this is safe.
- N argument must be a positive integer. items key is the string-encoded integer ("1"), not the type-prefixed ID.
