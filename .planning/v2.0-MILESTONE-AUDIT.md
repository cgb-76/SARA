## Critical Gap — MEET-01 Broken

**Requirement:** User can run `/sara-minutes N` to generate structured meeting minutes

**Break point:** `sara-update` Step 4 → `pipeline-state.json` extraction_plan

**Root cause:** `sara-update` writes `stage: "complete"` to `pipeline-state.json` but does not persist the real entity IDs (`assigned_id`) back into `extraction_plan` items. `sara-minutes` Step 2 reads `artifact.assigned_id` to reference created entities — this field is always `undefined`, producing broken entity references in all generated minutes.

**Files:**
- `.claude/skills/sara-update/SKILL.md` Step 4 — missing write-back
- `.claude/skills/sara-minutes/SKILL.md` Step 2 — reads non-existent field

**Fix options (choose one):**
1. `sara-update` Step 4: after creating each wiki page, update `extraction_plan[i].assigned_id` in `pipeline-state.json` with the real ID before writing `stage: "complete"`
2. `sara-minutes`: read entity IDs from `wiki/index.md` (look up by title match) instead of from `extraction_plan[].assigned_id`
